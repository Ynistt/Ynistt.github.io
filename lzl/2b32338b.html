一、基础

1.1 java优点

  1. java是一门面向对象语言，然后面向对象有三大特性（封装、继承、多态）
  2. java可以跨平台运行，只要经过一次编译，在其他平台安装对应的jvm就可以到处运行
  3. 拥有许多的类库可以给开发人员使用
  4. 具有安全性和健壮性（具有恶意代码攻击的安全机制）
  5. 强大的生态社区，例如Spring
  6. 支持多线程

1.2 java基本数据类型和引用类型

1byte = 8bit

基本类型：byte（8bit）-128~127、short（16）、int（32）、long（64）、float（32）、double（64）、char（16）、boolean（8）

引用类型：接口、数组、类、String

1.3 接口和抽象类的区别

共同点：

  1. 都不能被实例化
  2. 可以拥有抽象方法

区别：

  1. 抽象类使用的是abstract关键字，接口使用interface关键字
  2. 抽象类可以有静态方法、常量和成员变量，而接口只能有常量
  3. 抽象类只能被单继承，一个类可以实现多个接口
  4. 抽象类可以有构造方法，接口没有构造方法

1.4 final关键字的作用

  1. final可以修饰类，变量、方法
  2. 被final修饰的类是不可以被继承的，例如String类，类不能被继承，所以它的方法也是不能被重写的
  3. 如果是基本类型被final修饰，则相当于是一个常量
  4. 如果是引用类型被修饰，则引用的地址是不变的，但所对应的值可以改变

1.5 static的用法

  1. static是静态的意思，可以修饰类、变量、方法、代码块，不能修饰构造器
  2. 使用static后，该类就为静态类，我们就可以使用类名+方法名的方式在其他类中直接调用并使用，不必要在创建这个静态类的实例

1.6 String类

  1. String类被final进行修饰，所以是不能被继承的
  2. String str = “666”；和String str = new String（“666”）; 前者是在常量池进行管理，后者会先判断常量池中是否有，没有则创建，并且会在堆中创建一个对象

1.7 String和StringBuffer和StringBuilder

  1. String是一个不可变的字符串（字符序列）。应为String类被final修饰。因为String的复用性高。线程安全
  2. 如果要使用可变的字符序列，就使用StringBuffer和StringBuilder
  3. StringBuffer是线程安全的，但是效率低于StringBuilder，高于String
  4. StringBuilder是线程不安全的，但效率最高

1.8 == 和equals的区别

  1. == 和 equals在比较基本类型时，都是比较他们的值是否相等，因为equal方法默认是使用 == 进行判断
  2. 如果比较的是引用类型时，则比较的是他们的地址
  3. 如果重写了equals，则比较的是他们的内容
  4. 重写equals后，也需要重写hashcode，因为之间具有联动的关系
  5. 为什么有联动的关系，因为有着两个对象相等，则他们的哈希值也是相等，hashcode相等的两个对象equals不一定相等，所以需要重写

1.9 自动装箱和自动拆箱

  1. 自动装箱即把基本类型转成包装类型，int->Integer,long->Long
  2. 自动拆箱则相反
  3. 自动装箱时，使用了valueof方法
  4. 拆箱则使用了xxxvalue

1.10 异常处理机制

一种是try-catch方式，一种是throw方式

1.11 重载和重写

  1. 重写是发生在父子类的，子类重写父类的方法。子类访问修饰符权限要大于父类，并且父类方法被private、final修饰，则不能重写
  2. 重载是一个类中方法名相同，但是参数列表不同

1.12 多线程

线程是程序执行的最小单元，一个进程可以拥有多个线程

每个线程之间共享程序的内存空间

多线程可以提高cpu的利用率，减少等待响应的时间

1.13 创建线程的方式

  1. 继承Thread类 重写 run方法
  2. 继承Runnable接口，实现run方法
  3. 继承Callable接口，实现call方法，会有返回值，可以抛出异常
  4. 线程池创建（ThreadPoolExecutor类）

1.14 线程状态

  1. 初始状态：被创建，但没调用start方法
  2. 可运行状态：等待操作系统的调度
  3. 阻塞状态
  4. 等待状态
  5. 超时等待状态
  6. 终止状态

1.15 Spring常用注解

1.声明bean注解

@Component：泛指各种组件

@Controller @Service @Repository

2.注入bean的注解

@Autowired：由Spring提供 按类型注入

@Resource @Inject 由jsr提供 按名称注入

3.配置类注解

@Configuration：声明该类为配置类

@Bean ，声明方法返回的值为一个bean

@ComponentScan，扫描

@Value注解

注入属性
